# Software Architecture Notes

# What is Software Architecture
Software architecture is the overall structure or blueprint of a software application. It shows how different parts of the system work together

# Frontend
The frontend is the part of the application users can see and interact with.

# Backend
The backend is responsible for processing requests, applying business logic, and communicating with the database

# API
An API allows the frontend and backend to communicate by sending requests and responses

# Database
A database stores information such as users, products, orders, and other application data

# Three-Tier Architecture
Presentation Layer (Frontend)
            |
Application Layer (Backend/API)
            |
Data Layer (Database)

# Client-Side
Client-side code runs in the user's browser and handles the user interface and interactions

# Server Side
Server-side code runs on the server and processes requests, manages business logic, and communicates with the database
