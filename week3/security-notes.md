# Security Notes

# Why should secrets never be committed to Git
Passwords, API keys and other sensitive information should never be committed to Git because anyone with access to the repository could misuse them.

# What is a .env File
A .env file stores sensitive configuration values, such as API keys and database connection strings, outside of the applicationcode

# What is a .gitignore File
A .gitignore file tells Git which files and folders should not be tracked.

# What is an API key
An API key is a unique identifier that allows an application to access an API.

# Strong Passwords
A strong password should be long, unique, and contain a mix of uppercase letters, lowercase letters, numbers, and special characters.

# Two-Factor Authentication (2FA)
Two-factor authentication adds an extra layer of security by requiring a second verification step in addition to a password
