# Testing Notes

# What is Software Testing
Software testing is the process of checking whether a program works correctly and meets its expected requirements. It helps developers find and fix problems before users experience them.

# Why is Testing Important
Testing is important because it helps ensure that software is reliable, accurate, and free of major bugs. It improves the quality of the application and gives users a better experience

# Manuel Testing
Manual testing is when a person checks an application by using it directly. For example, clicking buttons, entering information

# Automated Testing
Automated testing uses software or scripts to perform tests automatically. It is faster than manual testing and is useful for running the same tests repeatedly

# What is a Bug
A bug is an error or defect in a program that causes it to behave incorrectly or produce unexpected results

# What is Unit Testing
Unit testing is the process of testing one small part of a program, such as a single function or method, to make sure it works correctly by itself

# What does "Passed" Mean
A test has passed when the program behaves exactly as expected

# What does "Failed" Mean
A test has failed when the actual result is different from the expected result

# Calculator Testing Example
If I were testing a calculator application, I would check the following:
- Verify that 2 + 3 returns 5
- Verify that 10 - 4 returns 6
- Verify that dividing by 0 displays an appropriate error message instead of crashing the application
