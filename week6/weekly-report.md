# Week 7 Weekly Report – Git Depth & Code Review

## Tasks Completed

During Week 7, I completed the following activities:

* Learned how merge conflicts occur and practiced resolving them manually using Git.
* Used Git commands such as `git log`, `git log --oneline`, `git diff`, and `git stash` to inspect project history and manage changes.
* Practiced using `git rebase` to update a feature branch with the latest changes from the main branch.

## What I Learned
This week strengthened my understanding of advanced Git workflows used in collaborative software development. I learned how to:

* Resolve merge conflicts safely.
* Read and interpret commit history.
* Compare file changes before committing.
* Temporarily save uncommitted work using Git Stash.
* Rebase feature branches onto the latest main branch.
* Provide clear, respectful, and constructive code review feedback.

These skills have improved my confidence in using Git effectively when working with a team.


## Challenges Faced

One challenge I encountered was working with `git rebase`, particularly when Git reported uncommitted changes that prevented the rebase from starting. I learned how to use `git status` to identify the issue and either commit or stash changes before retrying the rebase.

I also spent time understanding the difference between `git merge` and `git rebase`, especially how each method affects a project's commit history.



## Most Valuable Skill
The most valuable skill I gained this week was learning how to resolve merge conflicts and use advanced Git commands to manage project history. I also developed a better understanding of how professional code reviews contribute to improving software quality and collaboration within a development team.
